# Security Policy

TrigGuard is a safety-critical, governance-first system.
Security issues are handled through responsible, private disclosure.

## Reporting a Vulnerability

If you discover a security vulnerability, **do not open a public GitHub issue**.

Please report it privately by emailing:

**security@trigguardai.com**

Include, if possible:
- A clear description of the issue
- Steps to reproduce
- The potential impact
- Any relevant logs or proof-of-concepts

## Disclosure Process

- Reports are reviewed privately.
- Valid issues are investigated and mitigated.
- Fixes are released responsibly.
- Public disclosure, if any, occurs only after mitigation.

We do not offer bug bounties at this stage.

## Scope

This policy applies to:
- This repository
- Build and CI/CD workflows
- Dependencies and configuration contained here

This policy does **not** cover:
- Feature requests
- Product guarantees
- User behavior or outcomes
- Clinical or mental health claims

## Safe Harbor

We will not pursue legal action against researchers who:
- Act in good faith
- Avoid privacy violations and data destruction
- Do not exploit vulnerabilities beyond proof-of-concept
- Report issues privately and responsibly
