# Security Policy

TrigGuard Protocol is a **governance-first** specification. Security issues are handled through **responsible, private disclosure**.

## Reporting a vulnerability

If you discover a security vulnerability, **do not open a public GitHub issue**.

Please report it privately by emailing:

**security@trigguardai.com**

Include, if possible:

- A clear description of the issue  
- Steps to reproduce  
- The potential impact  
- Any relevant logs or proof-of-concepts  

We aim to **acknowledge** valid reports within **48 hours** (business days) and work toward coordinated disclosure after mitigation.

## Disclosure process

- Reports are triaged privately.  
- Valid issues are investigated and mitigated.  
- Fixes are released responsibly.  
- **Public** disclosure, if any, occurs only after mitigation.

Do **not** disclose vulnerabilities publicly until they are resolved.

We do not offer bug bounties at this stage.

## Scope

This policy applies to:

- This repository (spec, docs, conformance, SDK package)  
- GitHub Actions workflows and dependencies **as shipped here**  

This policy does **not** cover:

- third-party deployments or hosted services (report to the operator)  
- generic feature requests or non-security bugs (use Issues when appropriate)  

## Safe harbor

We will not pursue legal action against researchers who:

- Act in good faith  
- Avoid privacy violations, data destruction, or service disruption  
- Do not extort or demand compensation under threat of disclosure  

---

For product/runtime security (private monorepo), use the same contact; specify **“protocol”** vs **“runtime”** in the subject line.
